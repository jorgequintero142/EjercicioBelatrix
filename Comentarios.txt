=== Comentarios ===
1. El método LogMessage deberia tener el estilo camel case, es decir
el nombre que se recomienda seria logMessage.

2.El constructor recibe muchos parámetros, lo cual puede ser engorroso
al momento invocar el método o para modificarlo. Se recomienda usar un DTO en
su lugar. Las variables no deben ser static.

3. La variable 'initialized' no se esta usando.

4. Se crean objetos para realizar operaciones  con archivos en base de datos
sin haber validado que se hayan requerido, con lo que pueden crear objetos 
innecesarios, desperdiciando memoria.

5. No se cierra la conexión a la base de datos.

6. No esta bien definido el lanzamiento de errores. Específicamente, cuando
se lee el archivo, cuando se realiza la conexión y cuando se ejecuta la 
consulta. Se recomienda realizar la programación en cada una de las partes del
código mencionadas.

7. La sentencia "messageText.trim();" no tiene ningun proposito. Se deben
revisar las lineas siguientes relacionadas con la validacion de longitud y valor
null.

8. La variable 'int t;' que se usa para definir el tipo del mensaje, 
no es mnemonica, y ademas los valores no indican a simple vista el tipo
 del mensaje. Se recomienda usar un ENUM.

9. En el metodo LogMessage, se pueden omitir los parametros de tipo boolean, puesto
que en el constructor ya se han definido tres variables que pueden ser usadas para 
las mismas validaciones.

10. No es necesario enviar un parametro HashMap, este se puede tener valores 
por defecto.
